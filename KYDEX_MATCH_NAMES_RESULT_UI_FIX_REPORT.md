# KYDEX Match Names And Ahmad Integrity Fix Report

Date: 2026-05-04

## Scope Completed

This work closed both the result-page display issue and the trust issue in single-token Ahmad screening.

Completed outcomes:

- Screening result cards now show the actual matched listed name first, using backend match data instead of ambiguous placeholders.
- Arabic fallbacks and simplified Arabic reason text were added for missing or technical fields.
- The screening UI now surfaces explicit evidence fields such as matched field, matched alias, matched token, and explainable match evidence.
- Backend screening was tightened so weak single-token alias-only evidence no longer surfaces unrelated records unless the primary listed name or another identifier also supports the match.
- Ahmad-family normalization was hardened so valid Ahmad and Ahmed variants still survive the stricter evidence gate.
