## الغرض

يساعد هذا الدليل مستخدمي المجلس على تنفيذ أهم خطوات العمل داخل KYDEX أثناء اختبار القبول الرسمي.

يوفر KYDEX في هذا الإصدار المعتمد:

- واجهة عربية وإنجليزية
- دعم العرض من اليمين إلى اليسار
- فحص محلي اعتمادا على القوائم الرسمية المستوردة
- إدارة الحالات والمراجعة والتصعيد
- إنشاء حزمة أدلة
- تصدير الخط الزمني للامتثال
- بوابة نشر معتمدة بحالة productionValid true

## قبل البدء

- افتح بوابة KYDEX المخصصة للاختبار.
- استخدم الحساب التجريبي المخصص لدورك.
- تأكد من أن المتصفح يعرض النص العربي بشكل صحيح.
- جهز اسما عربيا وعينة اسم بالإنجليزية للاختبار.

## 1. تسجيل الدخول

1. افتح صفحة تسجيل الدخول.
2. أدخل البريد الإلكتروني.
3. أدخل كلمة المرور.
4. اضغط تسجيل الدخول.
5. تأكد من فتح لوحة التحكم أو الصفحة المناسبة لدورك.

النتيجة المتوقعة:

- ينجح تسجيل الدخول.
- يظهر اسم المستخدم والدور.
- لا تظهر صفحات خارج صلاحيات المستخدم.

## 2. التبديل بين العربية والإنجليزية

1. ابحث عن زر تغيير اللغة في أعلى الصفحة.
2. اختر العربية.
3. تأكد من تحول الواجهة إلى العربية ومن اتجاه العرض إلى اليمين.
4. اختر الإنجليزية.
5. تأكد من رجوع الواجهة إلى الإنجليزية واتجاه العرض إلى اليسار.

النتيجة المتوقعة:

- تتغير العناوين والقوائم والحقول بحسب اللغة.
- يبقى التنسيق العربي واضحا وسليما.
- لا تضيع بيانات الصفحة عند تغيير اللغة.

## 3. نظرة عامة على لوحة التحكم

تعرض لوحة التحكم ملخصا تشغيليا سريعا.

راجع العناصر التالية:

- بدء فحص جديد
- آخر العمليات أو آخر طلبات الفحص
- الحالات التي تحتاج إلى إجراء
- مؤشرات المخاطر العالية أو الحالات المصعدة
- حالة تحديث مصادر البيانات

النتيجة المتوقعة:

- تفتح لوحة التحكم بدون أخطاء.
- تظهر الإجراءات الأساسية بوضوح.
- تبقى النصوص منسجمة مع اللغة المختارة.

## 4. فحص اسم

1. افتح صفحة فحص جديد.
2. أدخل اسم الشخص أو الجهة.
3. إذا توفرت البيانات، أضف تاريخ الميلاد والجنسية ورقم المستند.
4. اختر مصادر الفحص.
5. ابدأ الفحص.

أمثلة مقترحة للاختبار:

- اسم عربي
- اسم إنجليزي
- اسم بصيغة نقل صوتي مختلطة
- مثال بدون تطابق

النتيجة المتوقعة:

- يعرض النظام قرارا آليا مثل تطابق حقيقي أو تطابق محتمل أو تطابق مستبعد أو لا يوجد تطابق أو بيانات غير كافية.
- يظهر مستوى المخاطر.
- يوضح النظام سبب التطابق أو سبب عدم وجود تطابق.

## 5. استخدام استخراج البيانات من المستندات

1. افتح صفحة استخراج المستندات.
2. ارفع عينة المستند المدعوم.
3. راجع الحقول المستخرجة.
4. صحح أي قيمة عند الحاجة.
5. أكد البيانات المستخرجة.
6. تابع إلى الفحص.

النتيجة المتوقعة:

- تظهر الحقول المستخرجة للمراجعة البشرية.
- لا يبدأ الفحص قبل التأكيد.
- ترتبط نتيجة الفحص بمسار استخراج المستند.

## 6. عرض نتائج الفحص

بعد الفحص، راجع ما يلي:

- مستوى المخاطر
- القرار
- مستوى الثقة
- اسم المصدر
- نسخة المصدر
- الدرجة أو مؤشر تشابه المرشح
- هل يلزم التصعيد أم لا

النتيجة المتوقعة:

- تكون النتيجة مفهومة لغير المختصين تقنيا.
- يميز النظام بين عدم التطابق والتطابق المحتمل والمخاطر العالية والمخاطر الحرجة.

## 7. فهم شرح قرار المطابقة

شرح قرار المطابقة هو طبقة التفسير الرقابية داخل KYDEX.

راجع العناصر التالية:

- الاسم أو الاسم البديل المطابق
- دعم التطبيع العربي أو النقل الصوتي عند الحاجة
- المؤشرات المساندة مثل الجنسية أو رقم المستند
- سبب القرار الآلي ومستوى الثقة
- مساحة قرار المراجع إذا تم إنشاء حالة

النتيجة المتوقعة:

- يستطيع المستخدم فهم سبب النتيجة.
- يبقى الشرح واضحا بالعربية والإنجليزية.

## 8. إنشاء الحالات ومراجعتها

عندما تكون النتيجة HIGH أو CRITICAL يجب أن ينشئ KYDEX حالة أو يوجه المستخدم إلى مسار المراجعة.

التسلسل المعتاد:

1. افتح قائمة الحالات.
2. اختر الحالة.
3. راجع نتيجة الفحص والأدلة الداعمة.
4. أضف ملاحظات أو عين مراجع إذا كان دورك يسمح بذلك.
5. سجل قرار المراجع أو اللجنة.

القرارات الممكنة:

- Clear
- Request more information
- Escalate internally
- Prepare SIC package
- Reject or block

النتيجة المتوقعة:

- تظهر الحالات عالية المخاطر في مسار المراجعة.
- يبقى سجل الحالة محفوظا.
- يتطلب أي تجاوز أو قرار مراجعة تبريرا واضحا.

## 9. استخدام لوحة مصادر البيانات

افتح لوحة مصادر البيانات لمراجعة القوائم المستوردة.

تأكد من ظهور:

- OFAC SDN
- OFAC Consolidated
- UNSEC
- حالة المصدر وسجل النسخ

النتيجة المتوقعة:

- تفتح بطاقات المصادر وصفحات التفاصيل بشكل طبيعي.
- تبقى الترجمة العربية والإنجليزية منسجمة.

## 10. مزامنة OFAC و UNSEC

هذه الخطوة مخصصة للأدوار الإدارية أو الامتثالية المخولة فقط.

1. افتح لوحة مصادر البيانات.
2. ابدأ المزامنة الرسمية لـ OFAC أو UNSEC.
3. انتظر حتى تكتمل العملية.
4. راجع النسخة الجديدة وتقرير الإدخال.

النتيجة المتوقعة:

- تكتمل المزامنة بنجاح.
- تظهر نسخة جديدة من القائمة.
- يظهر تقرير الإدخال والحالة.

## 11. عرض السجلات المستوردة

هذه الخطوة مخصصة للأدوار المخولة فقط.

1. افتح أحد المصادر.
2. افتح صفحة السجلات.
3. ابحث بالاسم أو الاسم البديل أو الجنسية أو البرنامج أو رقم المستند.
4. افتح صفحة تفاصيل أحد السجلات.

النتيجة المتوقعة:

- يمكن البحث في السجلات.
- تعمل عمليات البحث العربية وعمليات النقل الصوتي كما هو متوقع.
- يظهر المحتوى الخام للمصدر عند توفره.

## 12. تفعيل النسخ أو أرشفتها

هذه الخطوة مخصصة فقط لـ SUPER_ADMIN و COUNCIL_ADMIN.

1. افتح سجل نسخ المصدر.
2. اختر نسخة.
3. قم بتفعيلها أو أرشفتها.
4. تأكد من تغير الحالة.

النتيجة المتوقعة:

- تتغير حالة النسخة بشكل صحيح.
- يمنع المستخدم غير المخول من تنفيذ هذه العملية.

## 13. تصدير حزمة الأدلة

هذه الخطوة مخصصة لـ SUPER_ADMIN و COUNCIL_ADMIN و COMPLIANCE_OFFICER.

1. افتح حالة قابلة للمراجعة.
2. أنشئ حزمة الأدلة.
3. افتح أحدث حزمة أدلة.
4. تأكد من وجود بيانات الفحص والمراجعة.

النتيجة المتوقعة:

- ينجح إنشاء حزمة الأدلة.
- يمنع الدور غير المخول من الوصول.

## 14. تصدير الخط الزمني للامتثال

عرض الخط الزمني متاح لأدوار المراجعة. تنزيل ملف التصدير متاح فقط للأدوار الإدارية.

1. افتح حالة.
2. افتح الخط الزمني للامتثال.
3. إذا كنت SUPER_ADMIN أو COUNCIL_ADMIN فقم بتنزيل ملف التصدير.
4. تأكد من فتح الملف بنجاح.

النتيجة المتوقعة:

- يظهر الخط الزمني للأدوار المخولة.
- يمنع تنزيل التصدير لغير الأدوار الإدارية.

## 15. تسجيل الخروج

1. افتح قائمة المستخدم.
2. اختر تسجيل الخروج.
3. تأكد من العودة إلى صفحة تسجيل الدخول.

النتيجة المتوقعة:

- تنتهي الجلسة الحالية.
- لا تعود الصفحات المحمية متاحة إلا بعد تسجيل الدخول من جديد.

## تذكير بالأدوار

- NOTARY: الفحص والوصول المحدود إلى الحالات
- COMPLIANCE_OFFICER: المراجعة والقرارات وإنشاء حزمة الأدلة
- COUNCIL_ADMIN: إدارة تشغيل المجلس ومصادر البيانات
- SUPER_ADMIN: التحكم التشغيلي الكامل داخل KYDEX
