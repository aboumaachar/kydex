# أداة تدقيق ثيم الكاتب العدل الفردي وإضافات WordPress


> حزمة عمل عربية مبنية على خلاصة نقاشات بيئة كتاب العدل وKYDEX، وتشمل: محرك التحقق، موقع مجلس كتاب العدل، صفحة/ثيم الكاتب العدل، إضافات الربط API، تحليل عرض JAB، والطرح المقابل القائم على خدمات Web/SaaS بدلاً من تطبيقات الهاتف.

> ملاحظة منهجية: هذه الوثائق هي أدوات تدقيق واستراتيجية ومواصفات أولية. لا تُعد رأياً قانونياً نهائياً، ويجب مراجعتها من مستشار قانوني مختص قبل اعتمادها تعاقدياً أو تنظيمياً.



## منهجية التقييم الموحدة

يعتمد التدقيق على سلم من 0 إلى 5:

| الدرجة | المعنى |
|---|---|
| 0 | غير موجود إطلاقاً |
| 1 | مذكور نظرياً فقط |
| 2 | موجود جزئياً دون اختبار أو توثيق |
| 3 | موجود ويعمل أساسياً |
| 4 | موجود ومختبر ومتكامل مع بقية النظام |
| 5 | موجود، مختبر، موثق، قابل للتدقيق، ومناسب للإطلاق |

قرار التدقيق:

- أقل من 2.5: غير قابل للاعتماد.
- من 2.5 إلى 3.5: يحتاج تحسينات جوهرية.
- من 3.5 إلى 4.5: قابل للتجربة المحدودة.
- من 4.5 إلى 5: قابل للإطلاق المنضبط.


## 1. الهدف

توفير أداة تقييم لكل موقع فردي لكاتب عدل، بحيث لا يكون مجرد صفحة جميلة، بل بطاقة مهنية رقمية قابلة للنشر، التحديث، والحجز/التواصل، مع إمكانية الربط بخدمات المجلس أو KYDEX عند الترخيص.

## 2. تدقيق الثيم

| البند | الدليل المطلوب | الدرجة |
|---|---|---|
| دعم RTL كامل | عرض عربي صحيح | 0-5 |
| Responsive desktop/mobile | لقطات شاشة | 0-5 |

| صندوق تعريف الكاتب | صورة، اسم، صفة، موقع | 0-5 |

| حماية النموذج | anti-spam/rate limit | 0-5 |
| سرعة التحميل | assets optimized | 0-5 |
| توافق WordPress | no fatal errors | 0-5 |

## 3. تدقيق المحتوى التجريبي

يجب أن يتضمن الثيم Demo Content قابلاً للتعديل:

- اسم الكاتب العدل.
- الصفة والمنطقة.
- ساعات العمل.
- الخدمات التوثيقية العامة.
- العنوان ورابط الخريطة.
- أرقام التواصل.
- نص مهني قصير عن الثقة والسرية والدقة.
- روابط المجلس.

## 5. متطلبات الأمن

- عدم تخزين API key نصاً مكشوفاً.
- عدم إرسال بيانات حساسة بالبريد الإلكتروني دون ضبط.
- إخفاء رسائل الأخطاء التقنية عن المستخدم النهائي.
- تفعيل reCAPTCHA أو rate limiting.
- تحديث الإضافات والثيم دورياً.

## 6. مؤشرات الخطر الأحمر

- ثيم يعتمد على ملفات خارجية غير موثوقة.
- نموذج تواصل دون حماية.
- صفحات لا تعرض العربية بشكل سليم.
- API key داخل JavaScript أمامي.
- عدم وجود طريقة سهلة لتغيير بيانات الكاتب.

## 7. قرار الجاهزية

يجب ألا يسلّم أي موقع فردي قبل أن يحقق 4/5 في: RTL، النموذج، الأمان، الأداء، والتحديث السهل.
